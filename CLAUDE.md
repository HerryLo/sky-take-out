## Project Overview

This is a Spring Boot-based food delivery backend system (外卖系统后端) called "sky-take-out-back". The system uses a multi-module Maven project structure with three main modules:

- **sky-server**: Main Spring Boot application containing controllers, services, mappers, and configuration
- **sky-pojo**: Data transfer objects (DTOs), entities, and value objects (VOs)
- **sky-common**: Common utilities, exceptions, constants, and shared components

## Technology Stack

- **Framework**: Spring Boot 3.2.12
- **Database**: MySQL with MyBatis 3.0.5
- **ORM**: MyBatis with camelCase mapping
- **Cache**: Redis with Lettuce connection pool
- **API Documentation**: Knife4j (Swagger UI replacement)
- **Authentication**: JWT tokens
- **Validation**: Spring Boot Validation
- **Pagination**: PageHelper
- **Build Tool**: Maven
- **Java Version**: 17

## Project Structure

```
sky-take-out-back/
├── sky-server/          # Main application module
│   └── src/main/java/com/liuheng/
│       ├── controller/  # REST controllers (admin and user packages)
│       ├── service/     # Business logic services
│       ├── mapper/      # MyBatis mappers
│       ├── entity/      # Database entities (generated by mappers)
│       ├── config/      # Spring configurations
│       ├── interceptor/ # Request interceptors (JWT, SQL error)
│       ├── aspect/      # AOP aspects (auto-fill)
│       └── annotation/  # Custom annotations
├── sky-pojo/            # DTOs, VOs, and entities
└── sky-common/          # Shared utilities and exceptions
```

## Common Development Commands

### Running the Application

```bash

mvn spring-boot:run -pl sky-server

# Build the project
mvn clean compile

# Package the application
mvn clean package -pl sky-server
```


## Key Architectural Patterns

### 1. Multi-Module Maven Structure
- Separation of concerns: main application, DTOs, and common utilities
- Clean dependency management through parent POM

### 2. Layered Architecture
- Controllers handle HTTP requests/responses
- Services implement business logic
- Mappers handle database operations
- Common layer for shared utilities

### 3. Global Result Wrapper
- All API responses are wrapped in `Result<T>` class
- Success responses: `code=1, data=...`
- Error responses: `code=0, msg=...`

### 4. JWT Authentication
- Separate JWT configurations for admin and user users
- Interceptor-based authentication
- Token validation on protected endpoints

### 5. Auto-Fill Pattern
- `@AutoFill` annotation marks fields for automatic population
- Aspect-based implementation for create/update timestamps
- Used in `Employee` and `User` entities

### 6. Pagination Support
- PageHelper integration for database pagination
- Consistent pagination response format in `PageResult`

## Important Configuration

### Database Configuration
- MyBatis maps underscores to camelCase
- SQL logging enabled (StdOutImpl)
- MySQL connection pooling

### Redis Configuration
- Connection pool: max 8 active, 8 idle
- Database: 0 (default Redis DB)
- Password protected

### JWT Configuration
- Admin and user separate secrets and TTL (2 hours)
- Token header name: "token"

## Naming Conventions

- Controllers are grouped by role: admin/* and user/*
- Response format: `Result<T>` with success/error codes
- DTOs for request data, VOs for response data
- Constants in sky-common package for shared values

## Development Tips

- Enable SQL logging in dev mode for debugging
- Knife4j documentation is Chinese language
- All database entities use Lombok for getters/setters
- Interceptors handle JWT validation and SQL error logging
